# pack-for-chatgpt.sh

Утиліта для створення disposable git-репозиторіїв для роботи з ChatGPT
(рев'ю, патчі, аналіз).

## Опис

Утиліта створює архів із мінірепозиторієм, який містить лише потрібні
файли.\
Це дозволяє уникнути проблем із кешуванням, конфліктами та зайвим
контекстом.

------------------------------------------------------------------------

## Режими

### full

Створює репозиторій з усіма tracked файлами.\
Опція `--include-untracked` додає untracked файли.

### slice

Створює репозиторій лише з вибраними файлами або директоріями.

### changed

Пакує лише змінені та нові файли.

### history

Копіює git-історію (обмежену або повну).

------------------------------------------------------------------------

## Приклади

    ./pack-for-chatgpt.sh full overview
    ./pack-for-chatgpt.sh slice fix-bug src/app.py
    ./pack-for-chatgpt.sh changed review
    ./pack-for-chatgpt.sh history investigate --depth 50

------------------------------------------------------------------------

## Вихід

Архів створюється у директорії:

    chatgpt-packs/

Ім'я файлу:

    chatgpt-pack-<mode>-<task>-<timestamp>.tar.gz

Якщо файл уже існує:

    chatgpt-pack-... (1).tar.gz
    chatgpt-pack-... (2).tar.gz

------------------------------------------------------------------------

## Структура архіву

    chatgpt-pack/
      .git/
      <project files>
      patch.base.sha256
      patch.meta.json
      CHATGPT_PACK_USAGE.md

------------------------------------------------------------------------

## Призначення

-   Генерація патчів через `git diff`
-   Безпечний обмін кодом
-   Мінімізація контексту
-   Відсутність кеш-конфліктів

------------------------------------------------------------------------

## Рекомендації

-   Використовуй `slice` для задач
-   Використовуй `changed` для рев'ю
-   Уникай `history` без потреби
-   Не передавай секрети

------------------------------------------------------------------------

## Нотатки

-   `patch.base.sha256` --- контроль цілісності
-   `patch.meta.json` --- метадані паку
-   disposable repo не є джерелом істини (крім history)
